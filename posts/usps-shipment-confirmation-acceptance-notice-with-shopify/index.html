USPS Shipment Confirmation Acceptance Notice (SCAN) with Shopify

25.05.2020 12:31

The basic problem

Recipients want to see that their package shipped when they look at usps.com

Dropping off multiple packages at the post office can mean a delay:

  • the packages are not scanned immediately
  • wait in line and then go through each package one by one

Things to get into order now

  • Do you have a good process for making sure your packages are bundled for End of Day pickup or drop off?
    • Fill up a box or other container with all items that have a shipping label affixed. Keep a separate box of items that pare packaged but don’t have a shipping label. Ideally you’ll set these up in an assembly line fashion, where you pick an item from your “needs shipping label” box when you’re ready to generate a shipping label, move them to your “being labled” area, and only put them in your “ready for post office” container when the label is affixed.

What is a USPS Shipment Confirmation Acceptance Notice (SCAN) form?

The most important feature is that it is a single bar code that has ALL items. Rather than scan each package one-by-one, a single bar code scan it all it takes.

Is this the right solution for me?

  • If you only mail a few items per day you probably don’t need this. On the other hand, if you’re mailing more than five packages a day it’s probably time.

How to use it in my workflow?

The idea is that as you genereate a shipping label for each package during the day as you would normally, but at the end of the day there is a “close out”… You generate a SCAN form that contains a bar code that has all of items to be delivered to the post office that day.

Shopify apps that can generate a SCAN form:

  • Shippo App will print USPS SCAN forms.
  • ShipStation App ShipStation Print End of Day form will print the SCAN form if the Provider is Stamps.com or Endicia).
  • PirateShip App will print a USPS end-of-day SCAN form.
  • ShippingEasy App can create a USPS SCAN Form
